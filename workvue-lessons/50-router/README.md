# ガードの仕組み
遷移する前に遷移していいサイトかをチェックする行為


Vue Routerにおけるガード（ナビゲーションガード）は、ページ遷移の「前・中・後」に割り込んで、その遷移を許可するか、キャンセルするか、あるいは別のページへ飛ばすかを判断する検問所のような仕組み

## 【グローバルガード】
router.js 全体
全てのページ遷移で実行。アプリ全体のログインチェックなど。

## 【ルート単位ガード】
routes 配列の定義内
特定のURLにアクセスした時だけ実行。管理者専用ページなど。

## 【コンポーネント内ガード】
index.vue など
その部品が呼ばれた時だけ実行。未保存の入力がある時の警告など。


## 【実行される順番】
beforeRouteLeave: 今いるページを離れていいか？（コンポーネント内）

beforeEach: そもそも新しい場所へ向かっていいか？（グローバル）

beforeEnter: その特定のルートに入っていいか？（ルート単位）

beforeRouteEnter: コンポーネントを描画していいか？（コンポーネント内）

afterEach: 遷移が無事に完了した（事後処理：解析タグの送信など）
